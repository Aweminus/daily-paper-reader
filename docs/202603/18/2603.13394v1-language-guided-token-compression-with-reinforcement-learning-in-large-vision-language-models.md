## 摘要
大语言视觉模型 (LVLMs) 由于处理大量视觉 Token 而产生巨大的推理成本。现有方法通常难以将渐进式视觉 Token 缩减建模为具有序列依赖性的多步决策过程，且往往依赖于缺乏针对复杂推理轨迹进行自适应优化的手工设计评分规则。为了克服这些局限性，我们提出了 TPRL，这是一个强化学习框架，通过与最终任务性能直接挂钩的语言引导序列优化来学习自适应剪枝轨迹。我们将视觉 Token 剪枝建模为具有显式状态转移的序列决策过程，并采用自监督自编码器将视觉 Token 压缩为紧凑的状态表示，以实现高效的策略学习。剪枝策略通过从演示中学习进行初始化，随后使用近端策略优化 (PPO) 进行微调，以共同优化任务准确性和计算效率。我们的实验结果表明，TPRL 在推理过程中去除了高达 66.7% 的视觉 Token，并实现了高达 54.2% 的 FLOPs 减少，同时保持了近乎无损的平均准确率下降（仅 0.7%）。代码已发布在 https://github.com/MagicVicCoder/TPRL。

## Abstract
Large Vision-Language Models (LVLMs) incur substantial inference costs due to the processing of a vast number of visual tokens. Existing methods typically struggle to model progressive visual token reduction as a multi-step decision process with sequential dependencies and often rely on hand-engineered scoring rules that lack adaptive optimization for complex reasoning trajectories. To overcome these limitations, we propose TPRL, a reinforcement learning framework that learns adaptive pruning trajectories through language-guided sequential optimization tied directly to end-task performance. We formulate visual token pruning as a sequential decision process with explicit state transitions and employ a self-supervised autoencoder to compress visual tokens into a compact state representation for efficient policy learning. The pruning policy is initialized through learning from demonstrations and subsequently fine-tuned using Proximal Policy Optimization (PPO) to jointly optimize task accuracy and computational efficiency. Our experimental results demonstrate that TPRL removes up to 66.7\% of visual tokens and achieves up to a 54.2\% reduction in FLOPs during inference while maintaining a near-lossless average accuracy drop of only 0.7\%. ### 2. 核心方法论：TPRL 框架
TPRL（Language-Guided Token Pruning with Reinforcement Learning）将 Token 剪枝建模为一个**马尔可夫决策过程（MDP）**，通过强化学习优化剪枝轨迹。其核心流程分为三个阶段：

*   **状态压缩（自监督自编码器）**：
    *   为了降低强化学习的训练负担，设计了一个轻量级的 Token 级自编码器。
    *   将原始高维视觉 Token（如 1024 维）压缩到低维潜在空间（如 64 维），仅用于剪枝策略网络的输入，不影响输入给 LLM 的原始特征。
*   **从演示中学习（LfD）**：
    *   **策略初始化**：直接从零开始训练强化学习效率极低。TPRL 首先利用现有的启发式剪枝方法（如 Vispruner）生成 50K 条高质量剪枝轨迹作为“演示数据”。
    *   **监督预训练**：通过行为克隆（Behavior Cloning）让策略网络初步掌握 Token 的重要性分布。
*   **强化学习微调（PPO 算法）**：
    *   **奖励函数设计**：结合了**任务准确性奖励**（基于下游任务得分的变化）和**效率奖励**（鼓励更高的剪枝率）。
    *   **序列决策**：策略网络基于当前保留的 Token 和文本查询，通过多步（T步）迭代决定哪些 Token 应该被移除。
    *   **策略网络架构**：采用共享的注意力模块（Shared Attention Module）来捕捉 Token 间的相互关系及与文本查询的相关性。

### 7. 优点与亮点
*   **任务导向**：将剪枝与最终的语言生成任务直接挂钩，而非单纯依赖视觉特征。
*   **序列化建模**：通过 MDP 建模，实现了比单次剪枝更精细的“渐进式过滤”。
*   **训练效率优化**：通过自编码器压缩状态空间和 LfD 预训练，巧妙解决了强化学习在处理大规模视觉输入时收敛难的问题。
*   **通用性**：实验证明该框架可无缝迁移至不同的 LVLM 架构（LLaVA 和 Qwen）。

### 8. 不足与局限
*   **训练成本**：虽然推理效率提升了，但引入了额外的三个训练阶段（自编码器、演示学习、RL 微调），增加了模型部署前的准备成本。

*   **实时性开销**：虽然策略网络很轻量，但在推理时仍需运行一次策略网络，对于极度追求极致延迟的场景，这部分开销仍需考量。
