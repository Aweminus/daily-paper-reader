## 摘要
基于大语言模型（LLM）的智能体在视觉语言导航（VLN）任务中展现出令人印象深刻的零样本性能。然而，大多数零样本方法主要依赖闭源 LLM 作为导航器，这面临着高昂的 token 成本和潜在的数据泄露风险。最近的研究尝试通过结合开源 LLM 与时空思维链（CoT）框架来解决这一问题，但与闭源模型相比仍有很大差距。在这项工作中，我们通过对导航过程的详细分析，发现了一个关键问题：导航失忆症（Navigation Amnesia）。这一问题导致了导航失败，并扩大了开源与闭源方法之间的差距。为了解决这个问题，我们提出了 HiMemVLN，它将分层记忆系统集成到多模态大模型中，以增强视觉感知召回和长期定位，从而缓解失忆问题并提高智能体的导航性能。在模拟和真实环境中的广泛实验表明，HiMemVLN 的性能几乎是现有开源最先进方法的两倍。代码可在 https://github.com/lvkailin0118/HiMemVLN 获取。

## Abstract
LLM-based agents have demonstrated impressive zero-shot performance in vision-language navigation (VLN) tasks. However, most zero-shot methods primarily rely on closed-source LLMs as navigators, which face challenges related to high token costs and potential data leakage risks. Recent efforts have attempted to address this by using open-source LLMs combined with a spatiotemporal CoT framework, but they still fall far short compared to closed-source models. In this work, we identify a critical issue, Navigation Amnesia, through a detailed analysis of the navigation process. This issue leads to navigation failures and amplifies the gap between open-source and closed-source methods. To address this, we propose HiMemVLN, which incorporates a Hierarchical Memory System into a multimodal large model to enhance visual perception recall and long-term localization, mitigating the amnesia issue and improving the agent's navigation performance. Extensive experiments in both simulated and real-world environments demonstrate that HiMemVLN achieves nearly twice the performance of the open-source state-of-the-art method. The code is available at https://github.com/lvkailin0118/HiMemVLN.

*   **核心问题**：开源大语言模型（LLM）在零样本视觉语言导航（VLN）任务中，性能远逊于 GPT-4o 等闭源模型。
*   **研究动机**：

    *   **导航失忆症（Navigation Amnesia）**：作者发现开源模型在导航中存在两种失忆现象：
        1.  **短期感知失忆**：难以识别已访问区域，导致陷入局部循环或冗余探索。
        2.  **长期规划漂移**：随着步骤增加，模型容易忘记全局指令约束，导致决策偏离目标。
*   **整体含义**：通过引入受神经生物学启发的“分层记忆系统”，提升开源模型在连续环境下的状态感知和长期定位能力。

### 2. 方法论
HiMemVLN 核心思想是将记忆分为**视觉主导的短期记忆**和**语义主导的长期记忆**。
*   **短期定位系统（Short-Term Localer）**：
    *   **视觉图记忆（Visual Graph Memory）**：构建在线更新的拓扑图 $G_t = (V_t, E_t)$。每个节点存储位置的 CLIP 视觉特征、访问步数和频率。
    *   **定位与更新**：通过余弦相似度匹配当前观测与历史节点。若相似度高于阈值则判定为“重访”，并使用动量更新节点特征；否则创建新节点。
    *   **软约束注入**：将重访统计和方向优先级转化为文本提示，引导 MLLM 优先探索未访问区域。
*   **长期全局系统（Long-Term Globaler）**：
    *   **全局模式提取**：在任务开始时，利用 MLLM 提取指令中的主方向、最终目标和导航模式。
    *   **原点追踪（Origin Tracking）**：显式维护“从哪来”（CameFrom）变量，增强智能体的方向感。
    *   **全局反思**：在每一步决策时，将全局模式和历史摘要注入 Prompt，校准局部动作与全局意图的一致性。
*   **整体流程**：感知（RGB-D）→ 路标预测（Waypoint Prediction）→ 记忆注入（分层记忆）→ MLLM 推理决策 → 低级动作执行。

### 6. 主要结论与发现
*   **性能飞跃**：HiMemVLN 在 R2R-CE 上的成功率（SR）达到 30%，几乎是之前开源最强方法 Open-Nav (16%) 的**两倍**。
*   **缩小差距**：其性能已接近甚至超过部分基于 GPT-4 的闭源导航方法（如 DiscussNav）。
*   **记忆的重要性**：消融实验证明，短期记忆有效减少了局部环路，长期记忆则显著提升了长程指令的执行稳定性。
*   **模型底座影响**：Qwen2-VL-72B 在导航推理任务中表现出最强的零样本能力。

### 7. 优点与亮点
*   **认知启发**：借鉴海马体记忆索引理论，将视觉空间定位与语义目标规划解耦，设计合理。
*   **部署友好**：针对开源模型优化，解决了隐私和成本问题，且无需昂贵的任务特定微调。
*   **鲁棒性**：通过视觉图记忆（Visual Graph Memory）弥补了纯文本描述在空间表达上的不足，有效缓解了“导航失忆”。

### 8. 不足与局限
*   **性能上限**：尽管提升显著，但与最先进的监督学习方法（如 BEVBert 的 60% SR）相比仍有较大差距。
*   **硬件门槛**：虽然使用了开源模型，但 72B 规模的模型仍需高性能 GPU（如 L40）支撑，在嵌入式端侧直接运行仍有挑战。
*   **依赖外部模块**：导航性能高度依赖于预训练的路标预测器（Waypoint Predictor）和物体识别模型（RAM/SpatialBot），若底层感知出错，记忆系统也难以纠正。
