## 摘要
将具有可验证奖励的强化学习（RLVR）扩展到多模态大语言模型（MLLMs）面临一个根本性挑战：它们的响应本质上交织着感知相关 token（用于定位视觉内容）和推理相关 token（用于构建推理链）。这些 token 类型实例化了不同但又相互依赖的能力——视觉定位和符号推理——这使得孤立的优化变得不足。通过 token 级别的实证分析，我们证明了仅优化感知或仅优化推理 token 的表现始终不如全面优化，这强调了它们内在的耦合性。为了解决这个问题，我们提出了一种即插即用的 Token 重加权（ToR）策略，该策略通过识别这两类关键 token 并在 RLVR 训练期间动态重新加权，显式地对这种相互依赖性进行建模。在现有方法（如 GRPO 和 DAPO）的基础上应用 ToR，在多个多模态推理基准测试中实现了持续的性能提升，在准确的视觉定位和连贯的推理方面均达到了最先进的性能。

## Abstract
Extending Reinforcement Learning with Verifiable Rewards (RLVR) to multimodal large language models (MLLMs) faces a fundamental challenge: their responses inherently interleave perception-related tokens, which ground visual content, with reasoning-related tokens, which construct reasoning chains. These token types instantiate distinct yet interdependent capacities -- visual grounding and symbolic reasoning -- making isolated optimization insufficient. Through token-level empirical analysis, we demonstrate that optimizing either perception- or reasoning-only tokens consistently underperforms full optimization, underscoring their inherent coupling. To address this, we propose a plug-and-play Token-Reweighting (ToR) strategy that explicitly models this interdependence by identifying critical tokens of both types and dynamically reweighting them during RLVR training. Applied on top of existing methods (e.g., GRPO and DAPO), ToR delivers consistent performance gains across multiple multi-modal reasoning benchmarks, achieving state-of-the-art performance with both accurate visual grounding and coherent reasoning.

### 1. 核心问题与研究动机
*   **核心问题**：在多模态大语言模型中，响应内容由“感知相关 Token”（用于定位视觉内容）和“推理相关 Token”（用于构建逻辑链）交织而成。
*   **研究动机**：现有的可验证奖励强化学习（RLVR）方法通常将感知和推理能力孤立优化（如仅针对思维链或仅针对视觉增强）。作者通过实证研究发现，感知与推理在 Token 级别是高度耦合的，仅优化其中一类 Token 会导致性能显著下降（如推理增强但视觉误判，或视觉定位准但逻辑混乱）。因此，如何平衡并协同优化这两类 Token 是提升 MLLM 推理能力的关键。

### 2. 方法论：Token 重加权 (ToR)
ToR 是一种即插即用的策略，核心思想是识别关键 Token 并动态调整其在强化学习损失函数中的权重。
*   **推理 Token 识别**：基于**预测熵（Predictive Entropy）**。高熵位置通常对应推理链中的关键决策点（Forking points）。选取熵值最高的前 $\alpha_r$ 比例的 Token。
*   **感知 Token 识别**：基于**视觉敏感度（Visual Sensitivity）**。通过计算“有图像输入”与“无图像输入（空占位符）”时 Token 对数概率的差值（Log-probability difference）来衡量。差值越大，说明该 Token 对视觉信息的依赖度越高。
*   **核心算法流程**：
    1.  在 Rollout 生成阶段，计算每个 Token 的熵和视觉敏感度。
    2.  根据预设比例（如 30%）筛选出推理和感知 Token 集合。
    3.  在策略梯度计算（如 GRPO 或 DAPO 算法）中，为这两类 Token 分别分配权重 $\gamma_r$ 和 $\gamma_p$（非选定 Token 权重设为 0）。
    4.  通过这种显式的重加权，模型被强制在减少推理不确定性的同时增强视觉定位。
