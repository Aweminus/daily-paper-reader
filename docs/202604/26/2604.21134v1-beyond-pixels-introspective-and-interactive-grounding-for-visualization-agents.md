## 摘要
视觉语言模型 (VLMs) 经常误读数值、产生细节幻觉，并混淆图表中的重叠元素。目前的方法仅依赖于像素解释，导致了“仅限像素瓶颈” (Pixel-Only Bottleneck)：智能体将交互式图表视为静态图像，从而失去了访问编码精确数值的结构化规范 (specification) 的机会。我们提出了内省式与交互式视觉定位 (IVG)，该框架结合了：(1) 基于规范定位的内省，通过查询底层规范获取确定性证据；(2) 基于视图定位的交互，通过操作视图来消除视觉歧义。为了在排除 VLM 偏见的情况下进行评估，我们推出了 iPlotBench，这是一个包含 500 个交互式 Plotly 图表、6,706 个二元问题及基准真相规范的基准测试集。实验表明，内省提高了数据重建的保真度，而与交互结合则实现了最高的问答准确率 (0.81)，在重叠几何图形上获得了 +6.7% 的提升。我们进一步在部署的智能体中演示了 IVG，这些智能体能够自主探索数据并与人类用户进行实时协作。

## Abstract
Vision-Language Models (VLMs) frequently misread values, hallucinate details, and confuse overlapping elements in charts. Current approaches rely solely on pixel interpretation, creating a Pixel-Only Bottleneck: agents treat interactive charts as static images, losing access to the structured specification that encodes exact values. We introduce Introspective and Interactive Visual Grounding (IVG), a framework that combines (1) spec-grounded introspection, which queries the underlying specification for deterministic evidence, with (2) view-grounded interaction, which manipulates the view to resolve visual ambiguity. To enable evaluation without VLM bias, we present iPlotBench, a benchmark of 500 interactive Plotly figures with 6,706 binary questions and ground-truth specifications. Experiments show that introspection improves data reconstruction fidelity, while the combination with interaction achieves the highest QA accuracy (0.81), with +6.7 % gains on overlapping geometries. We further demonstrate IVG in deployed agents that explore data autonomously and collaborate with human users in real time.

### 1. 核心问题与研究动机
*   **核心问题：** 现有的视觉语言模型在处理图表时存在**“仅限像素瓶颈”（Pixel-Only Bottleneck）**。它们将交互式图表视为静态图像，导致在面对重叠元素、微小数值差异或复杂趋势时，容易产生数值误读、细节幻觉和空间歧义。
*   **研究动机：** 现实中的图表（如 Plotly, Vega-Lite 生成的）是由结构化规范（Specification，如 JSON）渲染而来的。人类通过缩放、切换图例等交互行为来消除歧义，但目前的智能体却被限制在静态像素层面，忽略了底层“事实来源”的结构化数据。

### 2. 方法论：IVG 框架
论文提出了**内省式与交互式视觉定位（Introspective and Interactive Visual Grounding, IVG）**框架，包含两个核心机制：
*   **基于规范的内省（Spec-grounded Introspection）：** 允许智能体直接查询图表的底层 JSON 规范。通过提取精确的数据数组和属性，智能体可以进行符号化推理（如计算最大值），从而避免像素估算的误差。
*   **基于视图的交互（View-grounded Interaction）：** 提供一组工具（通过 MCP 协议实现），包括：
    *   `relayout`：缩放或平移坐标轴。
    *   `legendclick`：切换特定图层的可见性，解决重叠问题。
    *   `selected`：框选特定区域并返回该区域内的点数据。
*   **焦点上下文（Focal Context）：** 交互操作会产生具体的参数（如坐标范围、曲线索引），这些参数作为“搜索键”，引导智能体在庞大的 JSON 规范中精准定位相关数据，实现视觉注意力与结构化数据的桥梁连接。

*   **数据集（iPlotBench）：** 为了公平评估，研究者构建了 iPlotBench 基准测试。

### 6. 主要结论与发现
*   **内省提升保真度：** 直接访问 JSON 规范（+Intro）显著提升了图表重建的准确性（$S_{Data}$ 从 0.88 提升至 0.90）。

*   **协同效应：** “Full”配置在综合问答准确率上表现最佳（0.81）。当智能体同时拥有两种能力时，它们会优先使用内省来获取数值，仅在必要时进行视图交互，从而优化了工具调用的效率。
*   **模型门槛：** IVG 的收益随模型规模增长，通常需要 20B 参数以上的模型才能有效编排这些复杂的工具调用。

### 7. 优点与亮点
*   **突破瓶颈：** 首次明确定义并解决了可视化智能体中的“仅限像素瓶颈”。
*   **确定性评估：** 摆脱了模糊的像素指标或不可靠的 VLM 评分，通过 JSON 结构比对实现了确定性的评估。
*   **实用性强：** 提出的 MCP 工具集可以直接集成到现有的 AI 助手（如 Claude Desktop, OpenHands）中。
*   **多场景应用：** 展示了 IVG 在实时人机协作、自主数据报告生成和机器学习超参搜索中的巨大潜力。

### 8. 不足与局限
*   **库的依赖性：** 目前主要针对 Plotly 库实现。虽然理论可推广至 Vega-Lite 或 ECharts，但适配不同库的规范格式需要额外的工程工作。
