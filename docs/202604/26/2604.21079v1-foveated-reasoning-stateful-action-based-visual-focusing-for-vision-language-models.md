## 摘要
视觉语言模型受益于高分辨率图像，但视觉 token 数量的增加带来了高昂的计算开销。人类通过凹式视觉（foveation）来解决这一矛盾：粗略的视野引导“看哪里”，而选择性获取的高锐度证据则细化“想什么”。我们提出了 Foveated Reasoner，这是一个自回归视觉语言框架，它在单一解码轨迹中统一了凹式视觉与推理。该模型从低分辨率视图开始，仅在需要时触发凹式视觉，从选定区域检索高分辨率证据，并将其注入回同一解码轨迹中。我们采用两阶段流水线训练该方法：首先通过冷启动监督引导凹式视觉行为，随后通过强化学习共同提高证据获取能力和任务准确性，同时抑制平庸的“全看”方案。实验表明，该方法学习到了有效的凹式视觉策略，并在多个视觉语言基准测试中，在严格的视觉 token 预算下实现了更高的准确率。

## Abstract
Vision-language models benefit from high-resolution images, but the increase in visual-token count incurs high compute overhead. Humans resolve this tension via foveation: a coarse view guides "where to look", while selectively acquired high-acuity evidence refines "what to think". We introduce Foveated Reasoner, an autoregressive vision-language framework that unifies foveation and reasoning within a single decoding trajectory. Starting from a low-resolution view, the model triggers foveation only when needed, retrieves high-resolution evidence from selected regions, and injects it back into the same decoding trajectory. We train the method with a two-stage pipeline: coldstart supervision to bootstrap foveation behavior, followed by reinforcement learning to jointly improve evidence acquisition and task accuracy while discouraging trivial "see-everything" solutions. Experiments show that the method learns effective foveation policies and achieves stronger accuracy under tight visual-token budgets across multiple vision-language benchmarks.

### 论文总结：Foveated Reasoning (FoveateR)

#### 1. 核心问题与整体含义（研究动机和背景）
视觉语言模型（VLM）在处理高分辨率图像时面临严重的**分辨率-效率权衡**：高分辨率能提供细节，但视觉 token 数量的增加会导致自回归解码的计算和内存开销呈二次方增长。
现有的“视觉聚焦”方法存在缺陷：
*   **多轮迭代法（Multi-pass）：** 需要多次运行完整的 VLM，计算开销大且会中断推理状态（Hidden State）。
*   **文本驱动法（Text-grounded）：** 通过文本输出坐标（如 `[x,y,w,h]`），存在 token 冗余、格式脆弱（易解析失败）以及坐标量化误差等问题。
**研究动机：** 借鉴人类视觉的“凹式注视点”（Foveation）机制，实现一种**有状态的（Stateful）、基于动作的（Action-based）**视觉聚焦，在单一解码轨迹中动态获取高分辨率证据。

#### 2. 方法论：核心思想与技术细节
FoveateR 将解码过程建模为一个**部分可观测马尔可夫决策过程（POMDP）**：
*   **核心思想：** 模型从低分辨率全局图开始，在生成文本的过程中，根据当前的隐藏状态（Hidden State）决定是否需要“看细节”。
*   **双策略并行：**
    1.  **Token 策略 ($\pi_\theta$)：** 预测下一个文本 token 或特殊的触发符 `<fov>`。
    2.  **凹式聚焦策略 ($\pi_\phi$)：** 当触发 `<fov>` 时，一个轻量级 MLP 头部直接根据隐藏状态预测连续的坐标框（Box），无需通过文本输出坐标。
*   **有状态注入：** 获取的局部高分辨率特征被直接注入当前的 KV-cache 中，推理状态不中断，后续生成直接基于新证据。
*   **两阶段训练流水线：**
    1.  **冷启动监督（Coldstart SFT）：** 使用 VLM 生成的伪标签（推理步骤与对应区域框）引导模型学会使用 `<fov>` 标签。
    2.  **强化学习（RL）：** 采用 **GRPO（Group Relative Policy Optimization）** 算法。奖励函数结合了任务准确率、格式正确性，并引入了**区域正则化项（Regularization）**，惩罚过大的聚焦区域，防止模型通过“全看”来刷分（Reward Hacking）。

#### 3. 实验设计
*   **数据集：** 
    *   训练集：Visual CoT (438K), RefCOCO/+/g (321K), ScienceQA (6K)。
    *   评估集：Visual CoT Benchmark（包含 DocVQA, TextVQA, GQA, CUB 等 12 个子任务）、V* Bench（高分辨率感知测试）。
*   **对比方法：** 
    *   标准 VLM：Llava 1.5, Qwen2.5-VL, Sphinx。
    *   视觉聚焦方法：VisCoT (多轮法), UV-CoT, VPT (文本驱动法), DocThinker。

*   **训练时长：** 
    *   7B 模型冷启动阶段约需 3 天。
    *   RL 阶段（70K steps）约需 3-4 天。
*   **模型底座：** 基于 Qwen2.5-VL (3B 和 7B 版本)。

#### 5. 实验数量与充分性
*   **实验规模：** 涵盖了 12 个以上的视觉语言基准测试，涉及文档理解、通用问答、关系推理和细粒度分类。
*   **消融实验：** 
    *   对比了“仅聚焦”与“交替推理”的监督效果。
    *   测试了不同正则化系数 $\lambda_{reg}$ 对 token 预算的影响。
    *   进行了**测试时干预实验**（如随机化坐标框、限制聚焦次数），验证了学习到的聚焦策略的有效性。
*   **评价：** 实验设计非常充分，通过跨领域的 benchmark 验证了模型的泛化能力，且对比了多种不同技术路线的基座，具有较高的客观性和公平性。

#### 6. 主要结论与发现
*   **高效性：** FoveateR 在仅使用约 300 个视觉 token 的情况下（远低于基准的 1152 个），在文档理解等任务上超越了需要完整高分辨率输入的模型。
*   **自适应性：** 模型学会了根据任务难度分配预算。简单物体识别通常不触发聚焦，而复杂的文档或细粒度对比则会触发多次聚焦。
*   **无需人工标注：** 证明了通过 VLM 自生成的伪标签进行冷启动，再配合 RL 优化，可以比使用人工标注的坐标框获得更好的性能。

#### 7. 优点
*   **架构优雅：** 将视觉聚焦动作集成在单一自回归流中，保持了推理状态的连续性。
*   **非语言化动作：** 避免了文本坐标带来的格式错误和 token 浪费。
*   **RL 驱动的效率：** 通过正则化项成功训练出“按需查看”的行为，实现了极佳的精度-效率平衡。

#### 8. 不足与局限
*   **内存增长：** 随着聚焦次数增加，KV-cache 会线性增长，对于极长序列的推理可能存在显存压力。
*   **动作空间简单：** 目前仅支持矩形框聚焦，不支持多区域同时聚焦或非矩形区域。
*   **应用限制：** 目前主要在静态图像上验证，尚未扩展到视频领域（视频中的时空冗余更适合此机制，但复杂度更高）。
*   **推理延迟：** 虽然 token 数减少了，但动态注入新 token 可能会对现有的批处理（Batching）优化带来实现上的挑战。
