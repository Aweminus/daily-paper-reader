## 摘要
最近，视觉 Token 剪枝已被用于处理多模态大语言模型（MLLMs）中大量的视觉 Token。然而，我们观察到，虽然现有的剪枝方法在简单的视觉理解任务上表现可靠，但在有效泛化到复杂的视觉推理任务时却面临困难，这是以往研究中尚未充分探索的关键差距。通过系统分析，我们发现解码过程中的相关视觉信息偏移（RVIS）是导致失效的主要原因。为解决这一问题，我们提出了解码阶段偏移感知 Token 剪枝（DSTP），这是一个无需训练的附加框架，使现有的剪枝方法能够在解码阶段将视觉 Token 与不断变化的推理需求相对齐。大量实验表明，DSTP 显著缓解了剪枝方法在复杂推理任务中的性能下降，同时在视觉理解基准测试中也持续带来了性能提升。此外，DSTP 在多种先进架构上均表现出有效性，突显了其泛化能力以及极低计算开销下的高效性。

## Abstract
Recently, visual token pruning has been studied to handle the vast number of visual tokens in Multimodal Large Language Models. However, we observe that while existing pruning methods perform reliably on simple visual understanding, they struggle to effectively generalize to complex visual reasoning tasks, a critical gap underexplored in previous studies. Through a systematic analysis, we identify Relevant Visual Information Shift (RVIS) during decoding as the primary failure driver. To address this, we propose Decoding-stage Shift-aware Token Pruning (DSTP), a training-free add-on framework that enables existing pruning methods to align visual tokens with shifting reasoning requirements during the decoding stage. Extensive experiments demonstrate that DSTP significantly mitigates performance degradation of pruning methods in complex reasoning tasks, while consistently yielding performance gains even across visual understanding benchmarks. Furthermore, DSTP demonstrates effectiveness across diverse state-of-the-art architectures, highlighting its generalizability and efficiency with minimal computational overhead.

*   **研究背景**：多模态大语言模型（MLLM）在处理高分辨率图像时会产生海量视觉 Token，导致巨大的计算和内存开销。现有的视觉 Token 剪枝方法（如 FastV）主要在**预填充（Prefill）阶段**根据初始注意力权重剔除冗余 Token。
*   **核心问题**：研究发现，现有的剪枝方法在简单的视觉问答（VQA）中表现良好，但在复杂的**视觉推理（VMR）**任务（如数学、逻辑推理）中性能大幅下降。
*   **研究动机**：作者通过诊断分析发现，复杂推理需要模型在解码过程中不断变换关注的视觉区域。这种现象被定义为**相关视觉信息偏移（RVIS, Relevant Visual Information Shift）**。静态剪枝在初始阶段丢弃了后续推理步骤所需的 Token，导致了推理失败。

### 2. 方法论：DSTP 框架
作者提出了**解码阶段偏移感知 Token 剪枝（DSTP）**，这是一种无需训练的插件式框架，包含两个核心模块：
*   **核心思想**：在解码过程中动态监测注意力变化，一旦检测到 RVIS，就从预留的 Token 池中检索并恢复当前相关的视觉信息。
*   **关键技术细节**：
    1.  **RISD（相关视觉信息偏移检测）**：在解码的每一步，计算当前 Token 对视觉区域的注意力向量与预填充阶段初始向量的余弦相似度。当相似度低于设定阈值 $\tau$ 时，判定发生了 RVIS。
    2.  **CPTS（上下文保留视觉 Token 交换）**：
        *   **重新评估**：利用当前查询（Query）对所有原始视觉 Token（包括已丢弃的）重新计算重要性得分。
        *   **Token 交换与合并**：选取当前最相关的 Top-K 个 Token，并将其与初始保留的 Token 集合取并集，形成一个临时的“上下文保留”集合。
        *   **持续生成**：该扩展集合会维持 $L$ 个解码步，以确保推理的连贯性，随后恢复到初始剪枝状态以保持效率。

### 6. 主要结论与发现

### 7. 优点
*   **揭示了本质原因**：首次系统性地识别并量化了 RVIS 现象，解释了为何视觉剪枝在推理任务中会失效。
*   **无需训练且易于集成**：作为一种推理侧的修改，不需要重新训练模型，具有极强的工程实用性。

*   **层级一致性限制**：目前 DSTP 将选定的 Token 统一应用于所有 Transformer 层，未考虑不同层对视觉信息需求的差异（早期层关注低级特征，深层关注高级语义）。
