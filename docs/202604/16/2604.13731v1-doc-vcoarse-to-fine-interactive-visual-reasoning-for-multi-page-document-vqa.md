## 摘要
多页文档视觉问答（DocVQA）需要对长篇且视觉密集的文档中的语义、布局和视觉元素进行推理。现有的无 OCR 方法在处理能力与精度之间面临权衡：端到端模型在文档长度增加时扩展性较差，而基于视觉检索的流水线则显得脆弱且被动。我们提出了 Doc-V*，这是一个无 OCR 的智能体框架，它将多页 DocVQA 建模为序列化的证据聚合过程。Doc-V* 从缩略图概览开始，随后通过语义检索和针对性的页面获取进行主动导航，并在结构化工作记忆中聚合证据以进行有据推理。通过专家轨迹的模仿学习进行训练，并利用群体相对策略优化（GRPO）进一步优化，Doc-V* 在回答准确性与证据寻找效率之间取得了平衡。在五个基准测试中，Doc-V* 的表现优于开源基准模型，并接近闭源商业模型，在域外性能上比 RAG 基准提升了高达 47.9%。其他结果表明，性能提升源于带有选择性注意力的有效证据聚合，而非单纯增加输入页数。

## Abstract
Multi-page Document Visual Question Answering requires reasoning over semantics, layouts, and visual elements in long, visually dense documents. Existing OCR-free methods face a trade-off between capacity and precision: end-to-end models scale poorly with document length, while visual retrieval-based pipelines are brittle and passive. We propose Doc-$V^*$, an \textbf{OCR-free agentic} framework that casts multi-page DocVQA as sequential evidence aggregation. Doc-$V^*$ begins with a thumbnail overview, then actively navigates via semantic retrieval and targeted page fetching, and aggregates evidence in a structured working memory for grounded reasoning. Trained by imitation learning from expert trajectories and further optimized with Group Relative Policy Optimization, Doc-$V^*$ balances answer accuracy with evidence-seeking efficiency. Across five benchmarks, Doc-$V^*$ outperforms open-source baselines and approaches proprietary models, improving out-of-domain performance by up to \textbf{47.9\%} over RAG baseline. Other results reveal effective evidence aggregation with selective attention, not increased input pages.

### Doc-V*: 针对多页文档视觉问答的由粗到细交互式视觉推理总结

#### 1. 核心问题与整体含义
*   **研究动机**：多页文档视觉问答（DocVQA）要求模型同时处理语义、布局和视觉元素。现有的无OCR方法面临两难境地：**端到端模型**（处理全量页面）受限于计算成本和“信息迷失”效应，难以扩展至长文档；**检索增强生成（RAG）**系统则是被动的，容易受到检索错误的影响，且缺乏多跳推理能力。
*   **核心问题**：如何模拟人类“先全局概览、后局部搜索、再整合证据”的阅读行为，实现高效且精准的长文档视觉推理？
*   **整体含义**：Doc-V* 提出了一种主动感知的智能体框架，将 DocVQA 视为一个序列化的证据聚合过程，通过交互式导航平衡了处理容量与推理精度。

#### 2. 方法论
*   **核心思想**：将多页 DocVQA 建模为**序列决策过程**。智能体不再一次性接收所有高分辨率页面，而是通过观察全局缩略图主动决定获取哪些页面的细节。
*   **关键技术细节**：
    *   **环境设计**：提供“全局缩略图概览”（Global Thumbnail Overview），将文档页面缩放并排列成网格，保留布局和标题等宏观信息，同时大幅降低初始视觉 Token 消耗（压缩比达 10-12 倍）。
    *   **动作空间**：定义了三种原子动作：`retrieval_page`（语义搜索）、`fetch_page`（按页码获取，用于查看邻近页或缩略图中的特定页）和 `answer`（输出最终答案）。
    *   **结构化推理（ReAct 风格）**：智能体输出遵循 `<think>`（包含分析、计划、总结）和 `<action>` 格式。
    *   **工作记忆（Working Memory）**：通过拼接每一轮的摘要，维护一个结构化的记忆流，防止长序列交互中的遗忘。
*   **训练流程**：
    1.  **SFT 阶段**：使用 GPT-4o 蒸馏出的高质量交互轨迹进行模仿学习。
    2.  **RL 阶段（GRPO）**：采用群体相对策略优化（Group Relative Policy Optimization），通过奖励函数（涵盖答案准确性、证据召回率和格式规范性）联合优化推理精度与搜索效率。

#### 3. 实验设计
*   **数据集/场景**：
    *   **域内（In-Domain）**：MP-DocVQA、DUDE。
    *   **域外（Out-of-Domain）**：SlideVQA（幻灯片）、LongDocURL（网页文档）、MMLongBench-Doc（超长文档，最高 468 页）。
*   **对比方法**：
    *   **端到端模型**：InternVL3, mPLUG-DocOwl2, DocVLM 等。
    *   **RAG 方法**：M3DocRAG, VisRAG, URaG 等。
    *   **智能体方法**：CogDoc, VRAG-RL。
    *   **闭源模型**：GPT-4o, Claude-3.7-Sonnet, Gemini-1.5-Pro。

#### 4. 资源与算力
*   **硬件环境**：使用了 **8 张 NVIDIA A100 (80GB) GPU**。
*   **模型底座**：基于 Qwen-2.5-VL-7B-Instruct。
*   **训练细节**：采用 BF16 混合精度。SFT 阶段训练 3 个 Epoch；GRPO 阶段在 2,048 个样本上训练 3 个 Epoch，每组采样 8 条轨迹（G=8）。

#### 5. 实验数量与充分性
*   **实验规模**：在 5 个主流基准测试上进行了全面评估，涵盖了从标准文档到超长文档的多种场景。
*   **消融实验**：针对全局缩略图、页面分析模块、工作记忆以及不同动作类型（检索 vs 获取）进行了详细的消融研究。
*   **鲁棒性分析**：测试了不同检索器（ColQwen, BM25, BGE）对性能的影响，以及交互步数和页面预算（K值）的敏感性。
*   **充分性评价**：实验设计非常充分且客观。通过与相同底座的 RAG 基线对比，清晰地证明了智能体策略的优越性；同时通过延迟和显存分析，客观评估了推理成本。

#### 6. 主要结论与发现
*   **性能卓越**：Doc-V* 在开源模型中达到 SOTA 水平，甚至在部分指标上超越了 GPT-4o 和 Claude-3.7-Sonnet。
*   **域外泛化强**：在 OOD 任务上表现突出，相比静态 RAG 基线提升高达 47.9%。
*   **证据聚合优于增加页数**：实验发现，长文档理解的关键在于**选择性注意力和有效的证据整合**，而非盲目增加输入页数（增加页数反而可能引入噪声）。
*   **纠错能力**：即使配合较弱的检索器（如 BM25），Doc-V* 也能通过主动的 `fetch_page` 动作补偿检索缺失，表现出极强的鲁棒性。

#### 7. 优点
*   **认知启发式设计**：借鉴“主动视觉”理论，由粗到精的策略非常符合人类处理复杂信息的逻辑。
*   **效率与精度的平衡**：通过缩略图和按需获取高分辨率 Token，在保持全局视野的同时控制了计算开销。
*   **训练创新**：成功将 GRPO 强化学习应用于多步视觉推理任务，有效提升了智能体的决策质量。

#### 8. 不足与局限
*   **底座依赖**：目前仅在 Qwen-2.5-VL 上验证，尚未探讨该框架在其他视觉语言模型底座上的通用性。
*   **场景限制**：主要针对单文档问答，尚未扩展到需要跨多个异构文档聚合信息的“多文档”场景。
*   **幻觉风险**：作为基于 VLM 的智能体，在处理极度模糊或复杂的视觉证据时，仍存在产生幻觉或推理错误的可能。
