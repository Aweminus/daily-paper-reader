## 摘要
将视觉语言模型扩展到视觉多智能体系统 (VMAS) 受到两个耦合问题的阻碍。首先，通信拓扑在推理前是固定的，使其无法感知视觉内容和查询上下文；其次，智能体的推理能力在部署期间保持静态。这些问题相互强化：僵化的拓扑结构无法利用更丰富的智能体专业知识，而静态智能体则缺乏针对特定查询进行专业化的动力。我们通过 SkillGraph 解决了这一问题，这是一个同时演化智能体专业知识和通信拓扑的联合框架。在该框架内，多模态图 Transformer (MMGT) 编码视觉标记、指令语义和活跃技能嵌入，以预测以查询为条件的协作图，从而用动态、内容感知的流信息取代手工设计的路由。与此相辅相成的是，技能设计器 (Skill Designer) 从失败案例中提取并细化推理启发式方法，构建一个自演化的多模态技能库 (Skill Bank)。至关重要的是，更新后的技能嵌入被反馈到 MMGT 中，使拓扑结构能够随着能力的增长而自适应。实验表明，SkillGraph 在四个基准测试、五种常见 MAS 结构和四个基础模型上均实现了一致的改进。代码可在 https://github.com/niez233/skillgraph 获取。

## Abstract
Scaling vision-language models into Visual Multiagent Systems (VMAS) is hindered by two coupled issues. First, communication topologies are fixed before inference, leaving them blind to visual content and query context; second, agent reasoning abilities remain static during deployment. These issues reinforce each other: a rigid topology fails to leverage richer agent expertise, while static agents lack incentives to specialize for a given query. We address this with SkillGraph, a joint framework that evolves both agent expertise and communication topology. Within this framework, a Multimodal Graph Transformer (MMGT) encodes visual tokens, instruction semantics and active skill embeddings to predict a query-conditioned collaboration graph, replacing hand-crafted routing with dynamic, content-aware information flow. Complementing this, a Skill Designer distills and refines reasoning heuristics from failure cases, constructing a self-evolving multimodal Skill Bank. Crucially, updated skill embeddings are fed back into the MMGT, enabling the topology to adapt alongside capability growth. Experiments show that SkillGraph achieves consistent improvements across four benchmarks, five common MAS structures and four base models. Code is available at https://github.com/niez233/skillgraph.

### 2. 方法论：核心思想与关键技术
SkillGraph 提出了一个**协同演化（Co-evolution）框架**，通过闭环机制同时优化“协作拓扑”和“智能体技能”。

*   **多模态图变换器 (MMGT)：**
    *   **核心思想：** 将拓扑设计视为一个受视觉和语义共同驱动的预测任务。
    *   **技术细节：** 
        1.  **多模态查询编码：** 使用 Cross-Attention 融合图像 Patch 和问题语义。
        2.  **智能体选择性视觉注意力：** 每个智能体根据其当前技能，通过门控机制（Gating Mechanism）独立关注图像的相关区域（如 OCR 技能关注文字区）。
        3.  **角色先验偏置：** 在 Graph Transformer 层中引入偏置矩阵，既保留人类设计的结构先验，又允许模型在证据充足时突破先验进行跨角色通信。
        4.  **边逻辑预测：** 通过双线性预测器生成有向边的概率，并利用 Bernoulli 采样生成有向无环图（DAG）。
*   **自演化技能库 (Self-Evolving Skill Bank)：**
    *   **技能定义：** 技能被定义为包含触发条件、推理策略、成功率记录和失败缓冲区的结构化元组。
    *   **技能设计器 (Skill Designer)：** 这是一个元智能体（Meta-agent），它定期诊断失败案例，执行“修改（Modify）”或“创建（Create）”操作，生成新的推理启发式规则。
*   **协同进化闭环：** 技能的更新会直接改变其嵌入向量（Embedding），这些向量作为 MMGT 的输入，驱动拓扑预测器自动调整路由策略，实现“能力增长带动结构优化”。

### 6. 主要结论与发现
*   **一致性提升：** SkillGraph 在所有基准和所有底座模型上均取得了性能提升，平均提升幅度在 1.2% 到 2.3% 之间。
*   **任务敏感性：** 在**数学推理 (MathVista)** 和**空间推理 (RealWorldQA)** 上提升最显著，而在 OCR 密集的 InfoVQA 上提升相对较小（受限于底层感知能力）。
*   **协同效应：** 消融实验显示，MMGT（动态拓扑）和技能演化（能力增强）结合的效果优于单一模块，证明了两者互为补充。
*   **收敛性：** 技能库的演化动作（创建/修改）随迭代次数增加而减少，表明系统能够收敛到一个稳定的专家知识库。

### 7. 优点与亮点
*   **内容感知拓扑：** 首次实现了将视觉特征直接引入多智能体通信拓扑的预测中，解决了“盲目路由”问题。
*   **闭环自进化：** 建立了一个无需人工干预的“失败诊断-技能更新-结构适配”闭环，赋予了系统终身学习的特性。
*   **灵活性：** 能够适配现有的任何多模态大模型，且对原始模型参数无侵入（主要通过 Prompt 和轻量级 MMGT 调整）。

### 8. 不足与局限
*   **感知瓶颈：** 实验显示，如果底层模型的视觉感知能力（如 OCR）达到极限，SkillGraph 的协作优化带来的边际收益会递减。
*   **计算开销：** 虽然 MMGT 很轻量，但多智能体系统本身的多次推理调用会显著增加推理延迟和 API 成本。
*   **技能库管理：** 随着迭代增加，虽然实验显示会收敛，但在极大规模任务场景下，如何防止技能库冗余或冲突仍需进一步探讨。
