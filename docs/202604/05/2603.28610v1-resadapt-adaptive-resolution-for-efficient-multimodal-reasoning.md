## 摘要
多模态大语言模型（MLLMs）通过提升输入保真度实现了更强的视觉理解能力，但由此产生的视觉 token 增长使得同时维持高空间分辨率和长时序上下文变得代价高昂。我们认为瓶颈不在于编码后的表示如何压缩，而在于编码器接收的像素量。为此，我们提出了 ResAdapt，这是一种输入端自适应框架，用于学习每帧在编码前应分配多少视觉预算。ResAdapt 将轻量级分配器（Allocator）与保持不变的 MLLM 主干网络相结合，使主干网络在接收经过算子转换的输入时，仍能保留其原生的视觉 token 接口。我们将分配问题建模为上下文老虎机（contextual bandit），并使用成本感知策略优化（CAPO）训练分配器，该方法将稀疏的 rollout 反馈转化为稳定的准确率-成本学习信号。在预算受限的视频问答、时序定位和图像推理任务中，ResAdapt 提升了低预算运行点的表现，且通常处于或接近效率-准确率前沿，在激进压缩下的推理密集型基准测试中增益最为显著。值得注意的是，在相同的视觉预算下，ResAdapt 支持多达 16 倍的帧数，同时带来超过 15% 的性能提升。代码可在 https://github.com/Xnhyacinth/ResAdapt 获取。

## Abstract
Multimodal Large Language Models (MLLMs) achieve stronger visual understanding by scaling input fidelity, yet the resulting visual token growth makes jointly sustaining high spatial resolution and long temporal context prohibitive. We argue that the bottleneck lies not in how post-encoding representations are compressed but in the volume of pixels the encoder receives, and address it with ResAdapt, an Input-side adaptation framework that learns how much visual budget each frame should receive before encoding. ResAdapt couples a lightweight Allocator with an unchanged MLLM backbone, so the backbone retains its native visual-token interface while receiving an operator-transformed input. We formulate allocation as a contextual bandit and train the Allocator with Cost-Aware Policy Optimization (CAPO), which converts sparse rollout feedback into a stable accuracy-cost learning signal. Across budget-controlled video QA, temporal grounding, and image reasoning tasks, ResAdapt improves low-budget operating points and often lies on or near the efficiency-accuracy frontier, with the clearest gains on reasoning-intensive benchmarks under aggressive compression. Notably, ResAdapt supports up to 16x more frames at the same visual budget while delivering over 15% performance gain. Code is available at https://github.com/Xnhyacinth/ResAdapt.

### 1. 核心问题与整体含义（研究动机和背景）
多模态大语言模型（MLLM）在处理视频和高分辨率图像时面临**“视觉 Token 爆炸”**的问题。现有的效率优化方案主要存在以下局限：
*   **模型端压缩（如 Token 合并/剪枝）：** 在编码后丢弃信息，无法恢复细粒度证据，且往往破坏推理引擎的优化内核。
*   **输出端智能体推理（如 迭代缩放/检索）：** 引入多轮交互延迟，且如果初始视图太模糊，可能会错过关键线索。
*   **核心观点：** 效率瓶颈不在于编码后的表示压缩，而在于编码器接收的**原始像素总量**。论文提出 **ResAdapt**，通过在编码前动态分配每帧的视觉预算（分辨率），实现“输入端自适应”。

### 2. 方法论
ResAdapt 的核心思想是将视觉资源分配建模为一个**上下文老虎机（Contextual Bandit）**问题。
*   **分配器（Allocator）：** 一个轻量级的预测模块（基于 SmolVLM 架构），接收粗糙的视觉特征和文本查询（Query），预测每帧的缩放因子 $s_t$。
*   **算子（Operator）：** 根据预测的 $s_t$ 对原始帧进行双线性插值缩放，随后送入主干 MLLM。
*   **成本感知策略优化（CAPO）：** 
    *   由于分配操作不可微且奖励稀疏，采用强化学习（RL）训练。
    *   **动态成本枢轴（Dynamic Pivot）：** 避免策略坍缩至最小预算，通过在固定目标和局部均值间插值来平衡准确率与成本。
    *   **非对称塑造（Asymmetric Shaping）：** 对高效且正确的回答给予适度奖励，对高耗能且错误的回答给予严厉惩罚。
*   **时序相似性正则化：** 引入正则项惩罚相邻相似帧的重复高预算分配，鼓励模型在时序上进行差异化分配。

### 3. 实验设计
*   **数据集/场景：**
    *   **视频问答（Video QA）：** VideoMME, LongVideoBench, MMVU, MLVU, VideoMMMU, LVBench。
    *   **时序定位（Temporal Grounding）：** Charades-STA, ActivityNet, NExT-GQA。

    *   **启发式方法：** 随机丢帧（Random Drop）、固定缩放（FixedScale）。
    *   **模型端压缩：** ToMe, VisionZip, FlashVid。
    *   **推理增强方法：** VideoAuto-R1。
*   **主干模型：** Qwen2.5-VL-7B 和 Qwen3-VL-8B。

*   **框架：** 基于 VeRL、DeepSpeed 和 vLLM 构建。

### 5. 实验数量与充分性
*   **实验规模：** 涵盖了从 32 帧到 128 帧的不同时序长度，测试了多个主流 MLLM 基准。
*   **消融实验：** 针对 CAPO 奖励设计、分布函数（Beta vs. Normal）、时序正则化项、以及算子泛化性（将缩放策略迁移至丢帧策略）进行了详细消融。
*   **客观性：** 实验在相同的视觉 Token 预算下进行对比，并绘制了效率-准确率帕累托前沿（Pareto Frontier），结果显示 ResAdapt 在激进压缩（如仅保留 10% Token）时优势最明显。

### 6. 主要结论与发现
*   **效率提升：** 在相同视觉预算下，ResAdapt 可支持多达 **16 倍**的帧数，性能提升超过 **15%**。
*   **主动感知：** 分配器学会了“主动感知”，将预算集中在信息密集帧（如包含文字、场景变换或关键动作的帧），而对冗余背景进行大幅压缩。
*   **推理密集型任务占优：** 在需要复杂推理的基准（如 VideoMMMU）上，ResAdapt 的增益远高于纯感知任务。
*   **硬件友好：** 由于不改变主干网络结构，ResAdapt 与 FlashAttention 等推理加速算子完全兼容。

### 7. 优点
*   **架构解耦：** 分配器与主干模型解耦，主干模型无需修改即可享受效率提升。
*   **查询感知：** 相比于传统的固定采样，ResAdapt 能根据问题动态寻找关键帧。
*   **训练稳定：** CAPO 解决了强化学习在处理成本约束时容易坍缩到极小值的稳定性问题。
*   **算子泛化：** 训练好的分配器可以零样本迁移到“帧选择”等其他输入端算子上。

### 8. 不足与局限
*   **固定开销：** 分配器本身引入了额外的计算开销（粗编码和预测），在处理极短视频（如少于 32 帧）时，这种开销可能抵消掉 Token 节省带来的收益。
*   **开环限制：** 分配决策是在推理开始前一次性完成的（开环），如果分配器错过了关键帧，后续推理阶段无法重新获取。
*   **跨域泛化：** 虽然在视频上表现优异，但在纯静态图像任务上的泛化表现参差不齐，有时甚至不如专门的图像压缩方法。
