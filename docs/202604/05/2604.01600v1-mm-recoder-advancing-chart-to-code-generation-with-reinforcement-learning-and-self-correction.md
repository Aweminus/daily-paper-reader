## 摘要
多模态大语言模型（MLLMs）近期在图表到代码生成等多模态编程任务中展现出了巨大的潜力。然而，现有方法主要依赖于监督微调（SFT），这要求模型通过图表-代码对学习代码模式，但并未让模型接触到代码执行环境。此外，尽管通过执行反馈进行自我修正是提高编程质量的一种潜在途径，但研究表明，即使是最先进的 MLLMs 在进行有效的自我修正时也面临困难。在这项工作中，我们介绍了 MM-ReCoder，这是一种通过强化学习（RL）训练并具备自我修正能力的图表到代码生成模型。我们提出了一种基于群体相对策略优化（GRPO）的两阶段多轮自我修正强化学习策略。第一阶段通过展开共享的第一轮对话来增强模型的自我修正能力，而第二阶段则通过全轨迹优化来提升编程能力。MM-ReCoder 通过与环境的交互以及迭代修正自身输出，学习生成更准确且可执行的代码。我们在三个图表到代码基准测试上的结果证明了 MM-ReCoder 达到了最先进的性能。

## Abstract
Multimodal Large Language Models (MLLMs) have recently demonstrated promising capabilities in multimodal coding tasks such as chart-to-code generation. However, existing methods primarily rely on supervised fine-tuning (SFT), which requires the model to learn code patterns through chart-code pairs but does not expose the model to a code execution environment. Moreover, while self-correction through execution feedback offers a potential route to improve coding quality, even state-of-the-art MLLMs have been shown to struggle with effective self-correction. In this work, we introduce MM-ReCoder, a chart-to-code generation model trained with reinforcement learning (RL) and equipped with self-correction ability. We propose a two-stage multi-turn self-correction RL strategy based on Group Relative Policy Optimization (GRPO). The first stage enhances the model's self-correction ability via rolling out a shared first turn, while the second stage improves the coding capability with full-trajectory optimization. MM-ReCoder learns to produce more accurate and executable code through the interaction with the environment and by iteratively correcting its own outputs. Our results on three chart-to-code benchmarks demonstrate the state-of-the-art performance of MM-ReCoder.

论文关注的是**图表到代码生成（Chart-to-Code）**任务，即输入一张科学图表图像，模型需生成可执行的 Python 代码以精确重现该图表。

*   **核心痛点**：即使是目前最强的模型（如 GPT-4o），在面对执行反馈时也难以进行有效的“自我修正”。通常模型在第二轮修正中分数的提升仅源于代码变得“可执行”，而对于原本已能运行的代码，其视觉还原度的修正往往是负向的。
*   **整体含义**：本文旨在通过强化学习（RL）让模型学会像人类一样通过“编写-执行-观察-修正”的迭代过程来提高代码质量。

#### 2. 方法论：核心思想与关键技术
MM-ReCoder 采用了“冷启动 + 两阶段强化学习”的训练框架：
*   **冷启动（Cold Start）**：
    1.  在 160k 图表-代码对上进行基础 SFT。
    2.  利用强模型（Qwen3-VL-235B）构造 7k 条高质量的两轮自我修正数据（通过拒绝采样确保第二轮比第一轮有显著改进），解决模型在 SFT 后丧失多轮对话能力的问题。
*   **两阶段多轮自我修正 RL（基于 GRPO 算法）**：
    *   **第一阶段：共享首轮优化（Shared First Turn）**。固定一个在线采样的第一轮输出，生成多个不同的第二轮修正候选，重点培养模型的纠错策略。
    *   **第二阶段：全轨迹优化（Full-Trajectory Optimization）**。同时对第一轮和第二轮的输出进行采样和联合优化，提升整体编码水平。
*   **奖励函数设计（Reward Design）**：
    *   **规则奖励（Rule-based）**：通过 Hook 拦截 Matplotlib 函数调用，对比图表类型、布局、文本内容和颜色。
    *   **模型奖励（Model-based）**：调用 Qwen2.5-VL-72B 作为评委，从图表类型、布局、文本、数据趋势、风格和清晰度六个维度打分。
    *   **格式奖励**：确保模型遵循 `<think>...</think>` 的思考链格式。

#### 6. 主要结论与发现
*   **性能领先**：仅 7B 规模的 MM-ReCoder 在 ChartMimic 上的表现优于许多参数量大得多的模型（如 Qwen3-VL-235B），并在多项指标上达到 SOTA。
*   **纠错能力**：MM-ReCoder 是首个在“已可执行代码”基础上实现正向视觉修正的模型。
*   **RL 策略**：先进行“共享首轮”训练再进行“全轨迹”训练是激发纠错能力的关键，直接进行全轨迹训练容易导致模型陷入“重复第一轮错误”的局部最优解。
*   **奖励模型**：模型奖励能有效解决规则奖励无法捕捉的“元素重叠”和“视觉清晰度”问题。

#### 7. 优点
*   **创新性**：首次将 GRPO 引入多轮多模态编程任务，并设计了针对性的两阶段训练策略。
*   **实用性**：解决了 MLLM 在代码生成中“拒绝修正”或“越改越错”的顽疾。
*   **效率**：证明了通过高质量 RL，小参数模型可以在特定垂直领域（图表编程）挑战超大规模通用模型。

#### 8. 不足与局限
*   **对齐风险**：定性分析发现，模型的思考链（Thinking Trace）与最终生成的代码有时并不完全一致（即思考对了但代码没改对）。
*   **泛化限制**：对于训练集中未涵盖的罕见图表类型，模型依然表现不佳。
*   **计算成本**：训练过程依赖高性能奖励模型（72B 级别）的实时推理，推理和训练的算力开销较大。
*   **轮次边际效应**：虽然支持多轮修正，但实验显示 4 轮之后性能提升基本停滞，甚至可能出现退化。
