## 摘要
多模态大语言模型（MLLMs）在视觉理解方面展现了强大的能力，但在需要深度搜索并将视觉证据与外部知识相结合的复杂多步推理中仍存在局限性。在这项工作中，我们通过为多模态深度搜索智能体构建高质量、经过验证的多跳视觉-语言训练数据来应对这一挑战。我们提出了一种用于基于证据的问答合成的多跳工具增强智能体（MTA-Agent），它能够自动选择工具及其参数，从视觉和文本源中检索并验证证据，并生成结构化的多跳问答轨迹。从多样化的 VQA 种子数据集出发，我们的流水线生成了一个大规模训练数据集 MTA-Vision-DeepSearch，包含 2.1 万个高质量的多跳示例。数据经过多阶段验证过程过滤，以确保事实一致性和答案的唯一性。利用 MTA-Vision-DeepSearch，一个 32B 的开源多模态搜索智能体实现了最先进的性能，在六个具有挑战性的基准测试中平均达到 54.63%，在相同的工具设置下优于 GPT-5 (51.86%)、Gemini-2.5-Pro (50.98%) 和 Gemini-3-Pro (54.46%)。我们进一步表明，在我们的数据上进行训练可以提高推理深度和工具使用行为，将平均步数从 2.27 增加到 4.28，并带来更系统、更持久的搜索策略。此外，我们证明了可以通过回放缓存的交互在不进行实时工具调用的情况下进行训练，从而显著降低训练成本。重要的是，我们将 MTA-Agent 作为多模态深度搜索的完全开放配方：我们发布了整个数据集、训练轨迹和实现细节，以实现可复现性并促进对开放多模态搜索智能体的未来研究。

## Abstract
Multimodal large language models (MLLMs) have demonstrated strong capabilities in visual understanding, yet they remain limited in complex, multi-step reasoning that requires deep searching and integrating visual evidence with external knowledge. In this work, we address this challenge by constructing high-quality, verified multi-hop vision-language training data for multimodal deep-search agents. We propose a Multi-hop Tool-Augmented Agent for Evidence-based QA Synthesis (MTA-Agent), which automatically selects tools and their parameters to retrieve and validate evidence from both visual and textual sources and generates structured multi-hop question-answer trajectories. Starting from diverse VQA seed datasets, our pipeline produces a large-scale training dataset, MTA-Vision-DeepSearch, containing 21K high-quality multi-hop examples. The data is filtered through a multi-stage verification process to ensure factual consistency and answer uniqueness. Using MTA-Vision-DeepSearch, a 32B open-source multimodal search agent achieves state-of-the-art performance, reaching an average of 54.63\% across six challenging benchmarks, outperforming GPT-5 (51.86\%), Gemini-2.5-Pro (50.98\%), and Gemini-3-Pro (54.46\%) under the same tool settings. We further show that training on our data improves both reasoning depth and tool-use behavior, increasing the average number of steps from 2.27 to 4.28, and leading to more systematic and persistent search strategies. Additionally, we demonstrate that training can be performed without real-time tool calls by replaying cached interactions, significantly reducing training cost. Importantly, we present MTA-Agent as a fully open recipe for multimodal deep search: we release the entire dataset, training trajectories, and implementation details to enable reproducibility and future research on open multimodal search agents.

*   **研究动机**：虽然现有的多模态大模型（如 Qwen、GPT-4V）在视觉理解上表现出色，但在处理需要结合外部知识、进行多步推理和深度搜索的复杂任务（如“图中车辆品牌的创始人死于哪座城市？”）时，往往受限于参数化知识的匮乏或推理深度的不足。
*   **核心挑战**：目前缺乏高质量、多样化且经过验证的多步视觉-语言搜索训练数据。现有的开源模型搜索步数短、策略单一，且数据合成过程中容易出现推理过程与答案不一致的问题。

### 2. 方法论：MTA-Agent 框架
*   **核心思想**：通过一个自动化的“智能体合成流水线”，将现有的单步视觉问答（VQA）种子数据转化为复杂的多跳（Multi-hop）深度搜索任务。
*   **关键技术流程**：
    1.  **种子筛选**：从 FVQA、OK-VQA 等数据集中筛选出必须依赖视觉信息且答案为特定实体（Proper Noun）的样本。
    2.  **QA 智能体合成**：使用一个基于 ReAct 框架的智能体，利用 **Web Search（网页搜索）、Web Reader（网页阅读）、Google Lens（视觉搜索）和 Image Search（图像搜索）** 四种工具收集证据。
    3.  **单步 QA 创建与验证**：在每一步搜索中生成新的问答对，并通过 GPT-5 进行多维度验证（事实性、唯一性、时间稳定性、实体依赖性）。
    4.  **多跳合并**：将多个单步问答链条合并为复杂的多跳问题，并进行“防泄漏检查”，确保不通过简单的单次搜索就能得到最终答案。
    5.  **强化学习训练**：使用 DAPO 算法对 Qwen3-VL 进行微调。
*   **创新点：交互回放（Replay）训练**：为了降低 RL 训练中调用搜索 API 的高昂成本，论文提出利用数据生成阶段缓存的工具调用记录进行“回放训练”，无需实时调用外部工具。

### 3. 实验设计
*   **数据集/基准测试**：在 6 个深度搜索基准上进行评估：MMSearch、MMSearch-Plus、BrowseComp-VL (BC-VL)、HR-MMSearch、FVQA 测试集以及自建的 **MTA-test**（经过人工校验和重写）。
*   **对比方法**：
    *   **闭源模型**：GPT-5、Gemini-2.5-Pro、Gemini-3-Pro（均在相同工具设置下对比）。
    *   **开源基座**：Qwen3-VL-8B/32B-Instruct。
    *   **专门的搜索智能体**：SenseNova-MARS、WebWatcher、MM-DeepResearch 等。

### 4. 资源与算力
*   **算力投入**：
    *   **32B 模型**：使用 **32 台 NVIDIA H100 GPU** 进行训练。
    *   **8B 模型**：使用 **8 台 NVIDIA H100 GPU** 进行训练。
*   **训练细节**：采用 verl-tool 框架，Batch Size 为 64，进行 100 个训练步（Steps），学习率为 2e-6。
*   **合成成本**：平均每个样本的 LLM 调用成本约为 **0.28 美元**，平均包含 9.3 次工具调用。

### 5. 实验数量与充分性
*   **实验规模**：合成了 **2.1 万条** 高质量多跳训练数据（MTA-Vision-DeepSearch）。
*   **消融实验**：对比了不同数据组合（仅新闻数据 vs. 全量数据）、不同跳数（2/3/4 跳混合）对性能的影响。
*   **人工验证**：对 444 条轨迹进行了人工评估，涵盖可理解性（97.7%）、事实性（89.4%）和视觉必要性（76.3%）。
*   **客观性**：实验涵盖了多个第三方主流基准，并指出了某些商业模型（如 Gemini-3-Pro）在特定基准上可能存在的训练数据污染问题，评价较为客观。

### 6. 主要结论与发现
*   **性能领先**：MTA-DeepSearch-32B 在六个基准上的平均准确率达到 **54.63%**，超越了 GPT-5（51.86%）和 Gemini-3-Pro（54.46%）。
*   **推理深度提升**：训练后，模型的平均搜索步数从 **2.27 步显著增加到 4.28 步**，展现出更强的持久性和系统性搜索行为。
*   **回放训练有效**：证明了使用缓存数据进行 RL 训练不仅能大幅省钱，且性能优于在简单数据集上进行实时工具调用的训练。

### 7. 优点
*   **完全开源**：提供了从数据合成流水线、数据集、训练轨迹到模型实现的全套“配方”，极大地推动了开源多模态智能体的发展。
*   **高质量数据保障**：通过多阶段验证和难度过滤（基于弱模型查询相似度），确保了合成数据的挑战性和准确性。
*   **训练效率优化**：提出的交互回放机制解决了智能体训练中 API 调用成本高、速度慢的痛点。

### 8. 不足与局限
*   **视觉必要性仍有提升空间**：人工验证显示约 23.7% 的问题在没有图像的情况下也能通过极深度搜索回答，虽然这有助于提升通用搜索能力，但削弱了“多模态”的特异性。
*   **依赖闭源模型进行合成**：数据生成和验证高度依赖 GPT-5 等顶级闭源模型，对于完全脱离闭源生态的开发者来说门槛较高。
*   **工具集固定**：虽然包含 7 种工具，但训练数据主要针对搜索类工具，对于更复杂的物理环境交互或特定软件操作的泛化能力未在文中深入探讨。
