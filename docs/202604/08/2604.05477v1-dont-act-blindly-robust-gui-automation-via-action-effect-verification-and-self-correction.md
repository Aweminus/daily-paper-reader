## 摘要
基于视觉语言模型（VLM）的自主 GUI 智能体通常假设环境响应是确定性的，在生成动作时并不验证之前的操作是否成功。在存在网络延迟、渲染延迟和系统中断的真实场景中，这种假设会导致未被察觉的动作失败、重复的无效行为以及灾难性的错误累积。此外，由于在线交互成本高昂且离线数据集缺乏实时反馈，学习鲁棒的恢复策略具有挑战性。我们提出了 VeriGUI（验证驱动的 GUI 智能体），它显式地建模了噪声环境下的动作结果和恢复过程。VeriGUI 引入了“思考-验证-动作-预期”（TVAE）框架来检测失败并引导纠错推理，并采用了一个两阶段训练流水线，结合了带有合成失败轨迹的鲁棒有监督微调（Robust SFT）以及带有非对称验证奖励的 GRPO。我们进一步基于 AndroidControl 构建了一个鲁棒性基准测试，用于评估失败识别和修正能力。实验表明，VeriGUI 在保持竞争力的标准任务性能的同时，显著减少了失败循环并提高了恢复成功率。

## Abstract
Autonomous GUI agents based on vision-language models (VLMs) often assume deterministic environment responses, generating actions without verifying whether previous operations succeeded. In real-world settings with network latency, rendering delays, and system interruptions, this assumption leads to undetected action failures, repetitive ineffective behaviors, and catastrophic error accumulation. Moreover, learning robust recovery strategies is challenging due to the high cost of online interaction and the lack of real-time feedback in offline datasets.We propose VeriGUI (Verification-driven GUI Agent), which explicitly models action outcomes and recovery under noisy environments. VeriGUI introduces a Thinking--Verification--Action--Expectation (TVAE) framework to detect failures and guide corrective reasoning, and a two-stage training pipeline that combines Robust SFT with synthetic failure trajectories and GRPO with asymmetric verification rewards. We further construct a Robustness Benchmark based on AndroidControl to evaluate failure recognition and correction. Experiments show that VeriGUI significantly reduces failure loops and improves recovery success while maintaining competitive standard task performance.

### 2. 方法论：TVAE 框架与两阶段训练
*   **核心思想**：构建一个闭环推理周期，将“预期效果”作为下一步“验证”的基准。
*   **TVAE 推理框架**：
    1.  **Thinking (T)**：结构化思考，包含验证、回顾、定位和动作规划。
    2.  **Verification (V)**：二分类判断（SUCCESS 或 NO_CHANGE），对比当前屏幕与上一步的预期。
    3.  **Action (A)**：生成可执行的 JSON 动作。
    4.  **Expectation (E)**：预测该动作执行后屏幕应有的视觉变化。
*   **两阶段训练流水线**：
    *   **阶段 1：Robust SFT（鲁棒性微调）**：在标准成功轨迹基础上，人工合成 30% 的“失败-恢复”轨迹（如模拟点击偏移导致屏幕不变），训练模型初步具备识别失败和生成恢复动作的能力。
    *   **阶段 2：V-A-E GRPO（强化学习优化）**：使用组相对策略优化（GRPO）。
        *   **隐式环境模拟**：利用 GUI 失败的“幂等性”（即失败通常导致屏幕不变），在离线数据上模拟环境反馈。
        *   **复合奖励函数**：包含动作奖励（$R_{act}$）、效果语义一致性奖励（$R_{eff}$）和**非对称验证奖励**（$R_{ver}$）。其中 $R_{ver}$ 严厉惩罚“明明失败却报告成功”的幻觉行为（惩罚权重为 -2.0）。

*   **闭环逻辑严密**：将预期（Expectation）作为显式的变量引入，解决了 VLM 智能体“断片式”操作的问题。
*   **低成本训练**：提出了一种利用离线数据模拟在线反馈的策略，避开了大规模在线 RL 训练时 Android 模拟器极高的基础设施需求。
*   **非对称奖励设计**：针对“幻觉”设置高额惩罚，是提升智能体自我监控能力的有效手段。

### 8. 不足与局限
*   **幂等性假设限制**：目前主要针对“动作后屏幕不变”的失败场景。对于那些“动作后跳转到了错误页面”或“产生部分渲染”的复杂非幂等失败，该框架的识别能力尚待验证。
*   **长程规划缺失**：虽然提升了步级的纠错能力，但对于需要极长步骤且涉及高层逻辑规划的任务，单纯靠步级验证仍难以完全解决。
*   **推理开销**：每一步多出的“思考”和“预期”Token 会增加首字延迟，对实时性要求极高的场景有一定影响。
