## 摘要
最近的进展将多模态大语言模型（MLLMs）的应用范围从标准的视觉问答扩展到了利用外部工具处理高级视觉任务。尽管取得了这些进展，但针对复杂任务精确执行并有效组合多种工具仍然是一个持续存在的瓶颈。受限于稀疏的工具集和简单的工具使用轨迹，现有的基准测试无法捕捉复杂且多样的工具交互，在评估实际现实场景下的模型性能方面存在不足。为了填补这一空白，我们推出了 VisualToolChain-Bench (VTC-Bench)，这是一个旨在评估 MLLMs 工具使用熟练程度的综合基准。为了与真实的计算机视觉流水线保持一致，我们的框架包含 32 种基于 OpenCV 的多样化视觉操作。这一丰富的工具集支持广泛的组合，使 VTC-Bench 能够严格评估多工具组合以及长程、多步计划的执行。为了进行精确评估，我们提供了 680 个精心策划的问题，这些问题分布在九个类别的认知层级中，每个问题都配有真实的执行轨迹。对 19 种领先的 MLLMs 进行的广泛实验揭示了当前模型在视觉智能体能力方面的关键局限性。具体而言，模型难以适应多样的工具集并泛化到未见过的操作，领先的模型 Gemini-3.0-Pro 在我们的基准测试中仅达到 51% 的准确率。此外，多工具组合仍然是一个持续的挑战。在面对复杂任务时，模型难以制定高效的执行计划，过度依赖于熟悉功能的狭窄、次优子集，而不是选择最优工具。通过识别这些基本挑战，VTC-Bench 建立了一个严格的基准，以指导更通用的视觉智能体模型的发展。

## Abstract
Recent advancements extend Multimodal Large Language Models (MLLMs) beyond standard visual question answering to utilizing external tools for advanced visual tasks. Despite this progress, precisely executing and effectively composing diverse tools for complex tasks remain persistent bottleneck. Constrained by sparse tool-sets and simple tool-use trajectories, existing benchmarks fail to capture complex and diverse tool interactions, falling short in evaluating model performance under practical, real-world conditions. To bridge this gap, we introduce VisualToolChain-Bench~(VTC-Bench), a comprehensive benchmark designed to evaluate tool-use proficiency in MLLMs. To align with realistic computer vision pipelines, our framework features 32 diverse OpenCV-based visual operations. This rich tool-set enables extensive combinations, allowing VTC-Bench to rigorously assess multi-tool composition and long-horizon, multi-step plan execution. For precise evaluation, we provide 680 curated problems structured across a nine-category cognitive hierarchy, each with ground-truth execution trajectories. Extensive experiments on 19 leading MLLMs reveal critical limitations in current models' visual agentic capabilities. Specifically, models struggle to adapt to diverse tool-sets and generalize to unseen operations, with the leading model Gemini-3.0-Pro only achieving 51\% on our benchmark. Furthermore, multi-tool composition remains a persistent challenge. When facing complex tasks, models struggle to formulate efficient execution plans, relying heavily on a narrow, suboptimal subset of familiar functions rather than selecting the optimal tools. By identifying these fundamental challenges, VTC-Bench establishes a rigorous baseline to guide the development of more generalized visual agentic models.

*   **研究动机**：多模态大模型（MLLM）正从被动的视觉问答向主动的“视觉智能体”演进，即通过调用外部工具解决复杂视觉任务。然而，现有基准测试（Benchmark）存在工具集稀疏（工具种类少）、调用轨迹简单（通常仅1-2步）等问题，无法反映真实世界中复杂的视觉处理流水线。
*   **核心问题**：如何系统性地评估 MLLM 在面对复杂视觉任务时，对多样化工具的**组合能力**、**长程规划能力**以及**执行精确度**。

### 2. 核心方法论
*   **VTC-Bench 框架**：构建了一个包含 **32 种基于 OpenCV 的视觉工具**的基准，涵盖几何变换、图像增强、特征提取和绘图四大模块。
*   **认知分层体系**：将 680 个问题划分为三个认知层级（Tier）和九个子类别：
    *   **Tier 1（感知增强）**：鲁棒 OCR、感知修复、注意力聚焦。
    *   **Tier 2（定量估计）**：测量、颜色分析、计数。
    *   **Tier 3（组合推理）**：图表分析、数学几何、空间推理。
*   **执行轨迹标注**：为每个问题提供了人工校验的**标准执行轨迹（Ground-truth Trajectories）**，不仅评估最终答案，还允许对中间规划步骤进行细粒度诊断。
*   **双重交互范式**：支持“代码驱动（Code-driven）”和“接口驱动（Interface-driven）”两种模式，评估模型生成 Python 代码或直接调用 API 的能力。

### 3. 实验设计
*   **评估对象**：涵盖了 **19 种领先的 MLLM**，分为四类：
    1.  原生工具使用模型（如 GPT-o3, GPT-o4-mini）。
    2.  闭源通用模型（如 GPT-5.2, Gemini-3.0-Pro/Flash, GPT-4o）。
    3.  开源工具专用模型（如 DeepEyes-v2, Thyme, V-Thinker）。
    4.  开源通用模型（如 Qwen3-VL 系列，规模从 8B 到 235B）。
*   **评估指标**：
    *   **APR (Average Pass Rate)**：平均通过率（核心指标）。
    *   **TCR (Tool Call Rate)**：工具调用率。
    *   **MAE (Mean Absolute Error)**：预测工具链长度与标准长度的平均绝对误差。
    *   **$Ef f_{tool}$ (Tool Usage Efficiency)**：工具使用效率，衡量有效步骤占总步骤的比例。

### 4. 资源与算力
*   **算力设备**：开源模型部署在 **NVIDIA H100 GPU** 上，使用 **vLLM** 框架进行推理。

### 5. 实验数量与充分性
*   **实验规模**：针对 19 个模型在三种设置（Base, Code, Interface）下进行了全面测试，总计处理了数千次推理任务。

*   **充分性评价**：实验覆盖了当前市面上最主流的闭源和开源模型，对比维度包括模型规模、训练背景和交互范式，实验设计客观且具有较强的说服力。

### 6. 主要结论与发现
*   **性能瓶颈**：当前最顶尖的模型在 VTC-Bench 上的表现依然欠佳，Gemini-3.0-Pro 以 **51.2%** 的准确率位居榜首，说明复杂工具链组合仍是巨大挑战。
*   **闭源 vs 开源**：闭源模型在引入工具后表现出显著的性能提升（如 GPT-4o 提升约 9.5%），而开源模型提升微小，甚至在某些任务中因工具误用导致性能下降。
*   **工具偏好与冗余**：模型倾向于使用少数“熟悉”的工具（如 Crop, Zoom, Rotate），在面对复杂任务时往往采用低效的“试错法”而非精确规划，导致工具链冗余度高（效率低）。
*   **感知是前提**：Tier 3 任务证明，强大的原生视觉感知能力是有效调用工具的前提。

### 7. 优点与亮点
*   **工具集丰富且专业**：引入 32 种 OpenCV 工具，比以往仅有 5-10 种工具的基准更贴近真实的计算机视觉工程实践。
*   **分层认知设计**：从基础修复到高级逻辑推理的任务设计，能够清晰地定位模型在哪个认知阶段出现失效。
*   **诊断性强**：通过提供标准轨迹和定义效率指标，能够量化分析模型在规划阶段的“幻觉”和低效行为。

### 8. 不足与局限
*   **样本规模**：680 个问题虽然质量高，但相比于某些数万量级的静态 VQA 基准，规模仍相对较小。
*   **评测成本**：由于涉及多轮交互和 API 调用，且部分评估依赖 LLM-as-a-Judge（GPT-4o），评测成本较高且存在潜在的评测偏见。
*   **动态环境缺失**：目前的工具调用仍基于静态图像处理，尚未扩展到视频流或实时交互的动态物理环境。
