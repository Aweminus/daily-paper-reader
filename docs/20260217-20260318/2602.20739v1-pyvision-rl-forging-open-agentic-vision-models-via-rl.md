## 摘要
智能体多模态模型的强化学习通常面临交互崩溃（interaction collapse）的问题，即模型会减少工具使用和多轮推理，从而限制了智能体行为的优势。我们推出了 PyVision-RL，这是一个针对权重开放多模态模型的强化学习框架，旨在稳定训练并维持交互。我们的方法结合了“过采样-过滤-排序”展开策略与累积工具奖励，以防止崩溃并鼓励多轮工具使用。利用统一的训练流水线，我们开发了用于图像和视频理解的 PyVision-Image 和 PyVision-Video。在视频推理方面，PyVision-Video 采用了按需上下文构建，在推理过程中选择性地采样与任务相关的帧，从而显著减少了视觉 token 的使用。实验表明，该模型具有强大的性能和更高的效率，证明了持续交互和按需视觉处理对于构建可扩展的多模态智能体至关重要。

## Abstract
Reinforcement learning for agentic multimodal models often suffers from interaction collapse, where models learn to reduce tool usage and multi-turn reasoning, limiting the benefits of agentic behavior. We introduce PyVision-RL, a reinforcement learning framework for open-weight multimodal models that stabilizes training and sustains interaction. Our approach combines an oversampling-filtering-ranking rollout strategy with an accumulative tool reward to prevent collapse and encourage multi-turn tool use. Using a unified training pipeline, we develop PyVision-Image and PyVision-Video for image and video understanding. For video reasoning, PyVision-Video employs on-demand context construction, selectively sampling task-relevant frames during reasoning to significantly reduce visual token usage. Experiments show strong performance and improved efficiency, demonstrating that sustained interaction and on-demand visual processing are critical for scalable multimodal agents.

### 1. 核心问题与研究背景
*   **核心问题**：多模态智能体（Agentic MLLMs）在进行强化学习（RL）微调时，普遍面临**“交互崩溃”（Interaction Collapse）**现象。即模型为了最大化短期奖励，会逐渐减少工具调用和多轮推理步数，退化为简单的单轮回答，导致其无法通过增加推理时计算量（Test-time Compute）来提升性能。
*   **研究背景**：虽然 OpenAI o3 等模型展示了工具使用在视觉理解中的潜力，但开源领域仍缺乏能稳定训练、支持长程交互且高效处理视频的智能体框架。

### 2. 方法论
*   **核心思想**：将 Python 解释器作为原始工具（Dynamic Tooling），通过改进 RL 的采样策略和奖励函数，诱导模型产生持续、有效的多轮交互行为。
*   **关键技术细节**：
    *   **过采样-过滤-排序（Oversampling-Filtering-Ranking）策略**：
        1.  **过采样**：生成比训练所需更多的 Rollout 轨迹。
        2.  **过滤**：剔除代码运行报错、超时或奖励方差为零（无学习信号）的无效组。
        3.  **排序（Standard Deviation Sorting）**：按组内奖励的标准差降序排列，优先选择难度适中、具有区分度的样本进行训练，起到类似课程学习的作用。
    *   **累积工具奖励（Accumulative Tool Reward）**：公式为 $R = R_{acc} + 0.1 \cdot n_{tc} \cdot 1\{R_{acc}=1\}$。只有在最终答案正确时，才根据工具调用次数（$n_{tc}$）给予额外奖励，从而鼓励模型进行深度探索而非盲目调用。
    *   **视频按需上下文构建（On-demand Context Construction）**：视频不预先采样输入模型，而是仅加载至 Python 环境。模型根据推理需要，通过编写代码动态抓取并观察特定帧。
    *   **改进的 GRPO 算法**：移除了优势估计中的标准差归一化项，以降低训练波动。
